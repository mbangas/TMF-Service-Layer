💼 Commercial Module Guide

The Commercial module implements the TM Forum pre-sales and contracting domain, covering TMF648 Quote Management and TMF651 Agreement Management. It closes the lifecycle loop: ServiceSpecification → ServiceQualification → Quote → Agreement → ServiceOrder.

1
Create a Quote (TMF648)
Generate a commercial quote linked to a ServiceSpecification

Click + New Quote from the main toolbar to open the creation form.

  • Name — a human-readable label for the quote (required).
  • Category — optional grouping (e.g., enterprise, residential).
  • Service Spec ID — link to a ServiceSpecification from the Catalog module.
  • Requested Completion Date — optional commercial deadline.

A new quote starts in the inProgress state.

POST /tmf-api/quoteManagement/v4/quote { "name": "Enterprise Fiber 1 Gbps", "category": "enterprise", "related_service_spec_id": "<spec-uuid>" }
2
Quote State Machine
Manage the quote lifecycle

Use the Transition button on any quote row to change its state:

FromToMeaning
inProgresspendingQuote submitted for review
inProgresscancelledCancelled before review
pendingapprovedInternally approved
pendingrejectedRejected — completion_date auto-set
pendinginProgressReturned for revision
approvedacceptedCustomer accepted — completion_date auto-set
approvedcancelledCancelled after approval
ℹ️
Terminal states are accepted, rejected, and cancelled. No further transitions are permitted.
3
Manage Quote Items
Add, view, and remove line items from a quote

Click Detail on any quote to expand its item list. Use + Add Item inline to append a new line item.

  • Action — add | modify | delete | noChange
  • Price — optional unit price (e.g., 150.00).
  • Price Type — recurring | nonRecurring | usage
  • Description — free-text line description.
POST /tmf-api/quoteManagement/v4/quote/{quote_id}/quoteItem { "action": "add", "item_price": 150.00, "price_type": "recurring", "description": "Monthly 1 Gbps port fee" }
4
Create an Agreement (TMF651)
Formalise a commercial or technical contract

Click + New Agreement from the Agreements tab toolbar.

  • Agreement Type — commercial | technical | SLA
  • Related Quote ID — link to the accepted Quote (optional).
  • Service Spec ID — link to the governing ServiceSpecification (optional).
  • Service Instance ID — link to the provisioned Service in Inventory (optional).
  • Start / End Dates — contract validity period.
POST /tmf-api/agreementManagement/v4/agreement { "name": "Enterprise Fiber SLA Agreement", "agreement_type": "SLA", "related_quote_id": "<quote-uuid>", "start_date": "2026-04-01T00:00:00Z", "end_date": "2027-03-31T23:59:59Z" }

6
Manage SLA Metrics
Add contractual SLA commitments to an agreement

Open an agreement detail panel and click + Add SLA. Each SLA metric defines a single contractual threshold.

  • Metric — availability | latency | throughput | mttr | packetLoss | jitter
  • Threshold — the contractual SLA target value (e.g., 99.9).
  • Unit — percent | ms | Mbps | hours
  • Conformance Period — daily | weekly | monthly
⚠️
Note: SLA metrics here are contractual commitments (per Agreement). Operational SLA monitoring thresholds live in the Assurance module (TMF657 SLOs) — they are distinct objects.
POST /tmf-api/agreementManagement/v4/agreement/{agreement_id}/agreementItem { "name": "Availability SLA", "metric": "availability", "metric_threshold": 99.9, "metric_unit": "percent", "conformance_period": "monthly" }
