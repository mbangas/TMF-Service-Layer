Sobre este guia

Este guia explica como criar e gerir Ordens de Serviço na TMF Service Layer (TMF641). O exemplo utilizado é a encomenda do serviço VM Completa — um bundle composto por três serviços elementares: VM Elementar vCenter, Gestão de Utilizadores e Passwords e Associação de Storage à VM.

O catálogo deve estar configurado antes de criar ordens. Consulte o Guia de Configuração do Catálogo para preparar as especificações, candidatos e categorias.

⚠
Pré-requisito — Catálogo Configurado
As especificações e categorias devem existir antes de criar ordens

Para criar uma Ordem de Serviço é necessário que existam no catálogo:

  • ServiceSpecification activa — define o tipo de serviço a encomendar (ex: "VM Completa")
  • ServiceCategory — categoriza a ordem para filtragem e organização
  • ServiceCandidate — publica a especificação no catálogo, tornando-a ordenável
ℹ️
Ainda não criou o catálogo? Utilize o Guia do Catálogo ou a secção Gerar Dados de Exemplo no final desta página para criar automaticamente as ordens de teste com base nas specs existentes.
🗺
Estrutura de uma Ordem de Serviço
Campos e relacionamentos do recurso ServiceOrder (TMF641)

Uma ServiceOrder agrupa um ou mais itens de ordem (OrderItem). Cada item referencia uma especificação de serviço e define a acção a executar.

Campos da ServiceOrder

Campo Obrigatório Tipo Descrição
namesimstringNome descritivo da ordem (ex: "Aprovisionamento VM Alpha")
descriptionnãostringDescrição detalhada do pedido
categorynãostringCategoria da ordem (deve corresponder a uma ServiceCategory existente)
prioritynãostringPrioridade de execução (ex: "1", "high", "normal")
requested_completion_datenãodatetimeData/hora limite pretendida para conclusão
order_itemnãoarrayLista de itens a encomendar (ver campos abaixo)

Campos de cada OrderItem

Campo Obrigatório Tipo Descrição
order_item_idsimstringIdentificador do item dentro da ordem (ex: "1", "2")
actionsimenumadd | modify | delete | noChange
service_spec_namenãostringNome da ServiceSpecification a ordenar (ex: "VM Completa")
service_namenãostringNome da instância de serviço a criar (ex: "vm-alpha-001")

Guia Passo a Passo

1
Criar uma Ordem de Serviço
TMF641 POST /serviceOrder

Uma ordem pode ser criada via interface ou API. O exemplo abaixo encomenda o bundle VM Completa para o Projecto Alpha.

1a — Ordenar o Bundle VM Completa (recomendado)

Esta é a forma mais comum: encomendar o bundle que inclui os três serviços elementares (VM, Utilizadores e Storage) com um único item de ordem. O motor de orquestração respeita as dependências de execução definidas nas especificações.

POST /tmf-api/serviceOrdering/v4/serviceOrder { "name": "Aprovisionamento VM Completa — Projecto Alpha", "description": "Provisionar VM Completa para equipa de desenvolvimento", "category": "Virtualização", "priority": "high", "order_item": [ { "order_item_id": "1", "action": "add", "service_spec_name": "VM Completa", "service_name": "vm-alpha-001" } ] } → Resposta: { "id": "uuid", "state": "acknowledged", … }

1b — Ordenar apenas um Serviço Elementar

Pode encomendar qualquer serviço elementar individualmente. Exemplo: adicionar apenas storage a uma VM já existente.

POST /tmf-api/serviceOrdering/v4/serviceOrder { "name": "Storage Extra — vm-alpha-001", "description": "Adicionar 200 GB SSD à VM existente", "category": "Virtualização", "order_item": [ { "order_item_id": "1", "action": "add", "service_spec_name": "Associação de Storage à VM", "service_name": "storage-vm-alpha-001" } ] }

1c — Ordem com múltiplos itens independentes

Numa mesma ordem pode incluir vários itens, cada um referenciando uma especificação diferente. Útil quando se quer aprovisionar várias VMs em simultâneo.

POST /tmf-api/serviceOrdering/v4/serviceOrder { "name": "Aprovisionamento VMs — Projectos Alpha e Beta", "order_item": [ { "order_item_id": "1", "action": "add", "service_spec_name": "VM Completa", "service_name": "vm-alpha-001" }, { "order_item_id": "2", "action": "add", "service_spec_name": "VM Completa", "service_name": "vm-beta-001" } ] }
✅
Na interface: Orders → botão "+ New Order". Preencha o nome, seleccione a categoria, seleccione a especificação no campo Service Spec Name e clique "Create Order". A ordem é criada no estado acknowledged.
2
Ciclo de Vida — Transições de Estado
TMF641 PATCH /serviceOrder/{id}

Após a criação, a ordem segue um ciclo de vida definido pelo TMF641. As transições de estado são feitas via PATCH na API ou pelos botões de acção na interface.

Avançar no ciclo de vida via API

-- Iniciar aprovisionamento -- PATCH /tmf-api/serviceOrdering/v4/serviceOrder/<order_id> { "state": "inProgress" } -- Marcar como concluída -- PATCH /tmf-api/serviceOrdering/v4/serviceOrder/<order_id> { "state": "completed" } -- Cancelar (só de acknowledged ou inProgress) -- PATCH /tmf-api/serviceOrdering/v4/serviceOrder/<order_id> { "state": "cancelled" }
✅
Na interface: use os botões Start, Complete, Fail e Cancel na linha de cada ordem na tabela. Os botões disponíveis mudam conforme o estado actual.
⚠️
Somente ordens no estado cancelled podem ser eliminadas. Para eliminar uma ordem, cancele-a primeiro e depois use o botão Delete.
3
Dependências entre Itens de Ordem
TMF641 OrderItemRelationship

Quando uma ordem contém vários itens elementares (em vez de um bundle), é possível declarar dependências entre itens dentro da mesma ordem. Isto garante que o motor de orquestração respeita a ordem de execução.

Exemplo: criar os três serviços elementares da VM Completa como itens separados, com dependências explícitas:

Criar a ordem com múltiplos itens

POST /tmf-api/serviceOrdering/v4/serviceOrder { "name": "VM Completa — itens explícitos com dependências", "order_item": [ { "order_item_id": "1", "action": "add", "service_spec_name": "VM Elementar vCenter", "service_name": "vm-alpha-001" }, { "order_item_id": "2", "action": "add", "service_spec_name": "Gestão de Utilizadores e Passwords", "service_name": "users-vm-alpha-001" }, { "order_item_id": "3", "action": "add", "service_spec_name": "Associação de Storage à VM", "service_name": "storage-vm-alpha-001" } ] }

Adicionar dependências entre itens (após criação da ordem)

-- Item 2 depende do Item 1 -- POST /tmf-api/serviceOrdering/v4/serviceOrder/<order_id>/orderItem/<item2_id>/orderItemRelationship { "relationship_type": "dependency", "related_item_label": "1" } -- Item 3 depende do Item 1 -- POST /tmf-api/serviceOrdering/v4/serviceOrder/<order_id>/orderItem/<item3_id>/orderItemRelationship { "relationship_type": "dependency", "related_item_label": "1" }
✅
Na interface: clique no botão 📋 Deps na linha da ordem. Na janela que abre, verá todos os itens da ordem. Use "+ Add Dep" em cada item para declarar a dependência, indicando o order_item_id do item de que depende (ex: "1").
✔
Resumo — Checklist para Criação de Ordens
Todos os passos por ordem, do catálogo à ordem concluída
  • 1 Verificar que existem ServiceSpecifications activas no catálogo (ex: "VM Completa")
  • 2 Verificar que existe uma ServiceCategory (ex: "Virtualização") para categorizar a ordem
  • 3 Navegar a Orders → + New Order
  • 4 Preencher o Name da ordem (obrigatório)
  • 5 Seleccionar a Category no dropdown
  • 6 Seleccionar a Service Spec Name no dropdown (ex: "VM Completa")
  • 7 Preencher o Service Name (nome da instância a criar, ex: "vm-alpha-001")
  • 8 Clicar "Create Order" — ordem criada no estado acknowledged
  • 9 Opcionalmente, usar 📋 Deps para declarar dependências entre itens
  • 10 Clicar Start para mover para inProgress
  • 11 Após o aprovisionamento, clicar Complete → estado completed
